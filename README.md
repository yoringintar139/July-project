# July-project
7월 아이디어 주제는 **사용자가 원하는 업무 환경에 맞는 직업/근무 형태를 추천해 주는 서비스**입니다.  
프로젝트 이름은 **직업 환경 추천자**입니다.

## 1. 문제 정의
- 많은 구직자/이직 준비자가 "무슨 직무를 할지"는 고민하지만, 실제로는 **어떤 환경에서 일할 때 성과와 만족도가 높은지**를 명확히 모르는 경우가 많습니다.
- 기존 채용 서비스는 주로 연봉, 회사명, 직무명 중심으로 정보를 제공해, 사용자의 성향(집중형/협업형, 재택 선호, 야근 허용도 등)을 반영하기 어렵습니다.
- 이 프로젝트는 사용자의 성향과 선호 조건을 질문 기반으로 수집하고, 이를 바탕으로 **적합한 업무 환경 + 직업군**을 추천해 의사결정을 돕는 것을 목표로 합니다.

## 2. 핵심 기능 (MVP)
- 사용자 성향 진단:
	- 총 12문항 내외의 진단 설문으로 사용자의 업무 적합 환경을 수치화합니다.
	- 진단 영역(예시):
		- 업무 방식: 자율적으로 일할 때 성과가 높은지, 명확한 지시가 있을 때 안정적인지
		- 협업 성향: 실시간 회의/대면 협업 선호도, 비동기 소통(메신저/문서) 선호도
		- 집중 환경: 오픈형 사무실/조용한 개인 공간 중 어느 환경에서 집중이 잘 되는지
		- 근무 조건: 원격/하이브리드/상주 근무 선호, 유연근무 허용 범위
		- 업무 강도 허용도: 마감 압박, 잦은 야근, 빠른 의사결정 환경 수용 가능성
	- 응답 방식:
		- 5점 Likert 척도(전혀 아니다 1점 ~ 매우 그렇다 5점)로 응답받아 정량화합니다.
		- 초기 MVP에서는 이해하기 쉬운 일반 문항 중심으로 구성합니다.
	- 점수 산출 방식:
		- 영역별 평균 점수를 계산하고 0~100으로 표준화합니다.
		- 최종적으로 "자율성", "협업 강도", "소통 방식", "근무 유연성" 4개 핵심 지표를 생성합니다.
	- 진단 결과 제공:
		- "나의 업무 환경 프로필" 한 줄 요약과 함께 강점/주의 포인트를 제공합니다.
		- 예: "독립 실행형 + 비동기 소통 선호 + 원격/하이브리드 환경에서 성과 우수"
- 원하는 환경 직접 설정:
	- 사용자가 본인이 원하는 근무 조건을 직접 선택할 수 있습니다.
	- 설정 항목(예시):
		- 근무 형태: 원격 / 하이브리드 / 상주
		- 협업 강도: 개인 중심 / 혼합 / 팀 협업 중심
		- 소통 방식: 비동기 중심 / 실시간 중심
		- 근무 시간: 고정 시간 / 유연 근무
		- 조직 문화: 안정형 / 성장형 / 도전형
	- 설정 방식:
		- 기본 선택값 중심으로 빠르게 입력할 수 있게 구성합니다.
- 업무 환경 프로필 생성:
	- 설문 결과(성향)와 사용자가 직접 설정한 원하는 환경을 함께 반영해 최종 프로필을 생성합니다.
	- 예: "자율성 높음 + 비동기 커뮤니케이션 선호 + 원격 근무 적합"
- 직업/직무 추천:
	- 환경 프로필과 매칭되는 직무를 우선순위로 제시합니다.
	- 각 추천 항목에 추천 이유를 함께 제공합니다.
- 결과 요약 리포트:
	- 사용자에게 한눈에 볼 수 있는 결과 카드(적합 환경, 추천 직무 TOP 3, 피해야 할 환경)를 제공합니다.

## 2-1. 부가 기능
- 맞춤 체크리스트:
	- 추천 직무 지원 전 준비 항목(기술, 포트폴리오, 자격요건)을 체크리스트로 제공합니다.
- 북마크/비교 기능:
	- 추천 결과를 저장하고 직무 간 비교할 수 있도록 지원합니다.
- 재진단 기능:
	- 시간이 지난 뒤 성향 변화를 반영해 다시 진단할 수 있습니다.
- 진단 신뢰도 보완:
	- 일부 문항을 역문항으로 구성해 일관성 없는 응답을 보정합니다.
	- 응답 시간 과도하게 짧음, 동일 선택 반복 등 품질 신호를 감지해 재응답을 권장합니다.
- 고급 조건 설정:
	- 필수 조건(반드시 지켜야 하는 항목)과 선호 조건(가능하면 맞추고 싶은 항목)을 구분해 입력합니다.
	- 우선순위(예: 근무 형태 > 근무 시간 > 소통 방식)를 지정할 수 있습니다.
- 고급 추천 필터:
	- 필수 조건 불일치 항목은 추천에서 제외하거나 낮은 순위로 처리합니다.

## 3. 기대 효과
- 사용자 관점:
	- 자기 성향에 맞는 진로를 빠르게 탐색해 시행착오를 줄일 수 있습니다.
	- "좋아 보이는 직무"가 아니라 "잘 맞는 근무 환경" 기반으로 선택이 가능해집니다.
- 서비스 관점:
	- 개인화 추천 경험을 통해 사용자 체류시간과 재방문율을 높일 수 있습니다.
	- 향후 채용 공고/교육 과정 추천으로 확장 가능한 기반을 확보합니다.

## 4. 핵심 화면 (웹)
- 온보딩/소개 화면:
	- 서비스 목적과 진단 흐름을 간단히 안내하고 시작 버튼을 제공합니다.
- 성향 진단 화면:
	- 질문 카드 형태로 답변을 수집하며 진행률(Progress Bar)을 제공합니다.
- 결과 화면:
	- 적합한 업무 환경 요약, 추천 직무 리스트, 추천 근거를 시각적으로 제공합니다.
- 상세 화면:
	- 각 직무의 주요 역할, 필요한 역량, 추천 이유를 상세히 보여줍니다.

## 4-1. 핵심 화면 부가 기능
- 결과 공유:
	- 결과를 링크/이미지 형태로 공유할 수 있습니다.
- 반응형 UI:
	- 모바일/태블릿/데스크톱에서 모두 읽기 쉬운 화면 구성을 지원합니다.
- 접근성:
	- 키보드 탐색, 명확한 대비, 스크린리더 친화적 텍스트를 고려합니다.

## 5. 개발 시 고려사항
- 추천 로직 설명 가능성:
	- "왜 이 직무가 추천되었는지"를 문장으로 설명해 사용자가 결과를 신뢰할 수 있어야 합니다.
- 데이터 구조화:
	- 사용자 응답, 환경 태그, 직무 태그를 일관된 스키마로 관리해야 합니다.
- 확장성:
	- 초기 MVP는 규칙 기반 추천으로 시작하고, 이후 사용자 피드백 데이터를 활용해 고도화할 수 있게 설계합니다.

## 5-1. 보안
- 개인정보 최소 수집 원칙:
	- 추천에 불필요한 민감 정보는 수집하지 않습니다.
- 전송/저장 보안:
	- HTTPS 사용, 비밀번호 해시 저장, 주요 데이터 암호화를 적용합니다.
- 인증/권한 관리:
	- 로그인 기능 도입 시 토큰 만료/재발급 정책과 기본적인 접근 제어를 포함합니다.

## 5-2. 기술 스택 (추천)
- 프론트엔드:
	- HTML/CSS/JavaScript 또는 React(빠른 UI 구성 및 상태 관리에 유리)
- 백엔드:
	- Node.js + Express (간단한 API 서버 구성)
- 데이터베이스:
	- SQLite(초기 개발) 또는 PostgreSQL(확장 단계)
- 배포:
	- Vercel(프론트) + Render/Railway(백엔드) 조합 또는 단일 플랫폼 배포

## 5-3. 핵심 기능 구현 포인트 (웹)
- 설문 점수화 로직:
	- 답변을 환경 태그 점수로 변환하는 규칙 테이블을 명확히 정의합니다.
- 추천 알고리즘:
	- 직무별 태그 가중치와 사용자 점수의 유사도를 계산해 순위를 산출합니다.
- 결과 시각화:
	- 막대 그래프/레이더 차트 등으로 "나의 성향"을 시각적으로 보여줍니다.
- 피드백 루프:
	- 사용자의 만족/불만족 피드백을 받아 추천 품질 개선에 반영합니다.

## 5-4. 디자인 방향
- 직관성 우선:
	- 텍스트 과다를 피하고 카드형 UI로 핵심 정보를 빠르게 전달합니다.
- 신뢰감 있는 톤:
	- 차분한 컬러 팔레트와 명확한 타이포그래피를 사용합니다.
- 동기부여 요소:
	- 결과 화면에 긍정적 메시지와 "다음 행동"(학습, 지원 준비)을 제시합니다.

## 6. 예상 일정 1개월(8시간)
- 1주차:
	- 요구사항 구체화, 화면 설계(와이어프레임), 데이터 구조 설계
- 2주차:
	- 설문 화면/결과 화면 프론트엔드 구현, 기본 API 구축
- 3주차:
	- 추천 로직 구현, 데이터 연동, 오류 처리 및 예외 케이스 보완
- 4주차:
	- UI 개선, 테스트(기능/반응형), 배포 및 최종 문서화

> 기준: 하루 8시간 집중 개발 기준.
